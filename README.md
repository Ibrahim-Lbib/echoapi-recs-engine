A high-performance, production-ready recommendation engine built with **FastAPI**, **SQLAlchemy**, and **Scikit-learn**. This system provides personalized recommendations based on user interactions using collaborative filtering.

---

## ✨ Features

- **🚀 Built for Performance**: Async API endpoints with FastAPI and `asyncpg`.
- **🔄 Event Tracking**: Log user interactions (`view`, `click`, `purchase`) in real-time.
- **🧠 ML-Powered RECS**: Item-item collaborative filtering based on cosine similarity.
- **🏗️ Background Training**: Dedicated worker and scheduler for model updates.
- **⚡ Two-Layer Caching**: Hybrid Redis + Postgres strategy for low-latency delivery.
- **🛡️ Production Ready**: Security middlewares, healthy checks, and non-root Docker execution.

---

## 🏗️ Architecture Overview

The system follows a modular architecture designed for scalability:

1.  **API Layer**: FastAPI handles incoming requests and event logging.
2.  **Service Layer**: Business logic for users, items, and recommendations.
3.  **ML Layer**: Similarity computation and model training logic.
4.  **Worker Layer**: Background tasks for periodic model retraining.
5.  **Infrastructure**: PostgreSQL for persistent data and Redis for ephemeral similarity scores.

## 🛠️ Quick Start

### 🐳 Using Docker (Recommended)

Run the entire cluster with a single command:

```bash
docker-compose up --build
```

### 🐍 Local Development

1. **Clone & Setup**:

    ```bash
    git clone https://github.com/Ibrahim-Lbib/echoapi-recs-engine.git
    cd echoapi-recs-engine
    python -m venv .venv
    source .venv/bin/activate  # On Windows: .venv\Scripts\activate
    pip install -r requirements.txt
    ```

2. **Configure Environment**:
    Copy `.env.example` to `.env` and fill in your values: `cp .env.example .env`

3. **Run Migrations**:

    ```bash
    alembic upgrade head
    ```

4. **Start Services**:

    ```bash
    uvicorn app.main:app --reload
    ```

## 📡 API Endpoints

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `POST` | `/api/v1/events/track` | Track a user interaction (view, click, purchase) |
| `GET` | `/api/v1/recommend` | Get personalized recommendations for a user |
| `POST` | `/api/v1/train/` | Trigger model retraining (Requires `X-API-Key` header) |
| `GET` | `/api/v1/health` | Check API, Database, and Redis health status |

## 🧠 ML & Recommendation Logic

The engine computes similarity between items based on a weighted interaction matrix:
- **Purchase**: Weight 3
- **Click**: Weight 2
- **View**: Weight 1

Cosine similarity is used to find "look-alike" items, and recommendations are generated by identifying items preferred by similar users that the target user hasn't interacted with yet.

---

## 🧊 Cold Start Handling

New users with no interaction history receive recommendations based on globally popular items, ranked by weighted interaction score (purchase×3 + click×2 + view×1). The response includes a `"cold_start": true` flag to indicate fallback results.
